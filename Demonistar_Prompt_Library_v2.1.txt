a novel or fictional world. Read the entire input internally
before producing any output. Do not summarize as you go.
Do not output anything until you have read everything.

As you read, tag internally any content falling into these
categories:

  World & Setting - geography, history, cosmology, rules of the
    world, how it operates physically or metaphysically.
  Characters - names, descriptions, relationships, motivations,
    backstory, voice, arc.
  Lore & Mythology - in-world belief systems, historical events,
    legends, cultural context.
  Plot & Story Structure - established story beats, conflicts,
    resolutions, timeline.
  Themes - core thematic intentions, philosophical underpinnings,
    authorial intent.
  Rules & Mechanics - any system with defined rules: magic,
    technology, social structure, economy, power dynamics.
  Unresolved Threads - things mentioned but not fully developed,
    open questions, flagged gaps.

After reading everything, produce a professionally formatted
Word document (.docx). Before selecting any colors or visual
styling, assess the tone, genre, and thematic content and derive
a cohesive color palette that reflects it. Apply that palette to
the title block, section headers, subheadings, callout boxes,
and dividers consistently throughout.

LSL SCRIPT DOCUMENT PROMPT:

Take the following Second Life LSL scripting project and create
a formal professional technical specification document in Word
(.docx) format. Use a clean corporate layout with a title block,
section dividers, and a color scheme in navy/steel metallics
appropriate to technical scripting work.

Required sections:
  1. Project Overview & Purpose
  2. Script Architecture (single/multi-script, linkset roles,
     HUD vs in-world object)
  3. Global Variables & Data Structures
     - All globals with type and purpose
     - LLD key schema (prefix + format)
     - List structures and their index layout
     - Notecard names and expected format
  4. State Machine & Events
     - All LSL states
     - Every event handled per state
     - Trigger conditions for each
  5. Function Reference
     - Every custom function: signature, purpose, parameters,
       return value, side effects
  6. Communication Protocols
     - llSay/llListen channel assignments
     - llRegionSayTo target logic
     - llInstantMessage usage and throttle awareness
     - llEmail if used
  7. Permissions & Asset Requirements
     - Animation permission requirements
     - Texture/sound UUIDs and full-perm requirements
     - Notecard read/write behavior
     - Copy/mod/trans requirements for distribution
  8. Known Constraints & Limitations
     - LSL-specific hard limits relevant to this script
     - Same-region-only functions called out explicitly
     - Dataserver async patterns noted
     - 64KB LLD cap awareness
  9. Edge Cases & Error Handling
     - Missing inventory items
     - Owner offline scenarios
     - Region restart / script reset behavior
     - Blacklist / whitelist edge cases if applicable
 10. Changelog
     - Table: Version | Date | Section | Change Summary | Reason
     - Prepopulate v1.0 with confirmed creation date
     - Mark deprecated approaches with [DEPRECATED] inline
 11. Conclusion & Roadmap
     - Next planned features
     - Known outstanding issues
